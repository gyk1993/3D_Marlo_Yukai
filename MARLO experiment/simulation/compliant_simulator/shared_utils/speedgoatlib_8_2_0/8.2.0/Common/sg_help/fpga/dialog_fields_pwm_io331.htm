IO331 > Driver configuration > PWM generation driver block

Dialog fields

Dialog fields

Tab: Module setup

 

 

Parameter

Description

FPGA module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

A vector of PWM channels this driver block entity will access. Channels can be specified in the range 1..N. The order of channels can be freely mixed, but the same channel shouldn't be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.

 

Tab: PWM generation

 

 

Parameter

Description

PWM pattern

The type of PWM signal.

There are two possibilities:

1.PWM is symmetric and centered on half the period

In this case only A On and B On are configurable. The driver will calculate A Off and B Off to keep the generation symmetric.

2.PWM is asymmetric

In this case A On and B On, and A Off and B Off are freely configurable.

PWM period vector

A vector of PWM period values in FPGA ticks.

The width of this vector must be equal to the width of the Channel vector parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Polarity of PWM A output

The PWM A output behavior between A On and A Off.

By default: PWM A is high between A On and A Off, but PWM A can be configured to be low between A On and A Off.

This parameter is useful when A and B must be complementary.

Polarity of PWM B output

The PWM B output behavior between B On and B Off.

By default: PWM B is high between B On and B Off, but PWM B can be configured to be low between B On and B Off.

This parameter is useful when A and B must be complementary.

Polarity of PWM Trigger output

Defines if a trigger is high (default) or low during trigger generation.

This parameter may be required if external hardware such as analog inputs of the IO106 have a different (low) polarity.

Trigger source

Defines the condition when a trigger signal is emitted.

Possible conditions are:

•PWM A On

•PWM A Off

•PWM A On and PWM A Off

•PWM B On

•PWM B Off

•PWM B On and PWM B Off

•Period starts

•Half the period

•Period starts and half the period

Trigger pulse length vector

The duration of a trigger pulse.

Possible values are between 0 and 255.

The width of this vector must be equal to the width of the channel vector parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

AB-output protection

This protection is used to prevent A and B being active at the same time.

Protection can be configured to be:

•Disabled: A and B can be high at the same moment

•A is forced off when B is on

•B is forced off when A is on

•Both A and B are disabled when A and B should be high at the same moment.

PWM update

By default, PWM parameters are updated with the given value each time a period starts. If required, PWM update can also be configured to occur at half the period or at both, the start of the period, and at half the period.
