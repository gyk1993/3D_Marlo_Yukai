IO901, 902 > Using shared memory to synchronize target machines

Run the models

The slave model should be loaded and started first (Model 2, Target 2). Once started, it waits for the interrupt from the master model.

 

 

Once the master model is loaded (Model 1, Target 1) and started, it sends interrupts to the slave target (Target 2).

 

This ensures that the two target machine work in sync, sharing the same variables. The execution time is exactly the same on both target machines.

 

 

When Target 1 application is stopped, application execution is also stopped on Target 2 stops too.
