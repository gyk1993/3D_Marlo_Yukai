Dialog fields

Tab: Module setup

 

Parameter

Description

FPGA module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

A vector of QAD channels this driver block entity will access. You can specify channels in the range 1..N. The order of channels can be freely mixed, but the same channel shouldn't be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Please observe that all the channels will have the same configuration. In the case you need to have a different configuration for others channels, you have to use an additional QAD setup block in your model.

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.

 

 

Tab: Quadrature decoder

 

Parameter

Description

Preload counter value vector

The counter value loaded when the model starts.

Operating mode

This QAD FPGA Code Module supports the following operating modes:
 

Mode        Resolution

Quadrature decoding        1x, 2x, and 4x
Direction count        1x, and 2x

Up/down count        2x, and 4x

Sampling

Sampling resolution.

 

Supported sampling resolutions are:

 

•1x (Quadrature decoding, and direction count only)

•2x (All modes)

•4x (Quadrature decoding, and up/down count only)

Index control mode

Defines how the QAD FPGA Code Module reacts at each low to high transition detected on the C/Input I/O line.

Supported modes are:

•Ignore index input: no action

•Load on index: Loads the 'Load on index value' parameter when the encoder index line goes high.

•Index reference mode: Counter is reset to zero at the first C/Index event detection and Index control mode then changes to “Ignore index input”

Counting mode

The counter range (min and max).

Options are:

•Disabled (Cycling counter): Counter runs from [-2’147’483’648 to + 2’147’483’647 (32-bit).

•Divide-by-N: If this mode is selected, parameter field 'Counting limit' appears to define the counter value range for a sampling resolution of one. Typical values are a power of two, such as 1024, 2048, 4096.

 

Note that if you set a resolution of 2 or 4, the counter value range will be automatically multiplied by this factor.

 

Example:

If you set a 'Counting limit' of 1024, and a sampling resolution of 1, the counter range is: [-1024 to + 1023].
 
If you set a 'Counting limit' of 1024, and a sampling resolution of 4, the counter range is: [-4096 to 4095].

Counting limit

This is enabled if the 'Counting mode' parameter is set to 'Divide-by-N'.

See 'Counting mode' parameter description for more  information.

Latch mode

Counter latch mode.

Options are:

•Disabled: No latching

•Latch on C/Index: Current counter value is stored into the position register (latched). The counter value will only be updated again at the next low to high transition event on the C/Index line.

•Hardware latch: latches counter value read by dedicated hardware input (D-line).

•Software latch: the model latches counter values for all channels configured with the specific QAD driver block.
