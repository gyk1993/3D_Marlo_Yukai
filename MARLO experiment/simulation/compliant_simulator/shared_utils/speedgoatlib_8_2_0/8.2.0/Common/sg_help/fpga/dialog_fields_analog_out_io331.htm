IO331 > Driver configuration > Analog > Analog output driver block

Dialog fields

Dialog fields

 

Parameter

Description

FPGA module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

Define a vector of DAC channels this driver block entity will access. You can specify channels in the range 1..N. The order of channels can be freely mixed. The same channel can’t be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Inter-module synchronization

This parameter defines if the model initiates the conversion. In this case the driver is an initiator.

The “DAC trigger” physical pin becomes an output which propagates this signal.

If the conversion is triggered by an external source, the driver is then configured as target. In this case the “DAC trigger” physical pin becomes an input used to read a trigger signal.

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.
