IO30X, IO31X > Driver configuration > PWM generation driver block

Input and output ports

Input and output ports

This driver block has no output ports and a maximum of 10 input ports, depending on which input ports are enabled in the block’s dialog box (see below). The 10 possible input ports are:

 

Input port

Description

Stop/Start

Sets the PWM generation mode.

An element value of 0 causes the PWM Generation to be put into Stopped mode the next time the PWM cycle completes. In addition, the A-Output and B-Output signals are set to the corresponding A-output Stop State input port element and B-output Stop State input port element values.

An element value of 1 causes the PWM Generation to be put into Started mode. In addition, the A-Output and B-Output signals are initialized to the corresponding A-output Start State input port element and B-output Start State input port element values.

This input port is of data type Boolean. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector' parameter.

A-output Start State

The level to set the A-Output signal on starting PWM generation.

An element value of 0 causes the A-Output signal of that channel to be initialized to Low the next time the PWM Generation is started.

An element value of 1 causes the A-Output signal of that channel to be initialized to High the next time the PWM Generation is started.

This input port is of data type Boolean. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector' parameter.

A-output Stop State

The level to set the A-Output signal on stopping PWM generation.

An element value of 0 causes the A-Output signal of that channel to be set to Low the next time the PWM Generation is stopped.

An element value of 1 causes the A-Output signal of that channel to be set to High the next time the PWM Generation is stopped.

This input port is of data type Boolean. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector' parameter.

B-output Start State

The level to set the B-Output signal on starting PWM generation.

An element value of 0 causes the B-Output signal of that channel to be initialized to Low the next time the PWM Generation gets started.

An element value of 1 causes the B-Output signal of that channel to be initialized to High the next time the PWM Generation gets started.

This input port is of data type Boolean. The width of this input port has to be equal to the number of PWM channels defined by the block’s 'Channel vector' parameter.

B-output Stop State

The level to set the B-Output signal on stopping PWM generation.

An element value of 0 causes the B-Output signal of that channel to be set to Low the next time the PWM Generation is stopped.

An element value of 1 causes the B-Output signal of that channel to be set to High the next time the PWM Generation is stopped.

This input port is of data type Boolean. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector parameter'.

A-compare Count

When the counter is equal to the value in this parameter the A-Output signal is toggled.

This input port is of data type UINT32. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector parameter'.

Each element can be in the range from 1 to 2^32-1.

B-compare Count

When the counter is equal to the value in this parameter the B-Output signal is toggled.

This input port is of data type UINT32. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector parameter'.

Each element can be in the range from 1 to 2^32-1.

Period-compare Count

This parameter defines half of the period for PWM generation. When the counter is equal to the value in this parameter the counting direction is changed from up to down.

 

This input port is of data type UINT32. The width of this input port must be equal to the number of PWM channels defined by the block’s 'Channel vector' parameter.

Each element can be in the range from 1 to 2^32-1.

A-action

This input port expects a matrix of size 4 rows x n columns where n equals the number of PWM channels by the block’s 'Channel vector' parameter. Each element can be in the range from 0 to 3. The element value defines the action for each event.

The action for the Zero-Event is defined as the first element in the respective column.

The action for the Period-Compare-Event is defined as the second element in the respective column.

The action for the A-Up-Compare-Event is defined as the third element in the respective column.

The action for the A-Down-Compare-Event is defined as the fourth element in the respective column.

This input port is of data type UINT8.

B-action

This input port expects a matrix of size 4 rows x n columns where n equals the number of PWM channels by the block’s 'Channel vector' parameter. Each element can be in the range from 0 to 3. The element value defines the action for each event.

Each element can be in the range from 0 to 3. The element value defines the action for each event.

The action for the Zero-Event is defined as the first element in the respective column.

The action for the Period-Compare-Event is defined as the second element in the respective column.

The action for the B-Up-Compare-Event is defined as the third element in the respective column.

The action for the B-Down-Compare-Event is defined as the fourth element in the respective column.

This input port is of data type UINT8.

 

Note: It is the responsibility of the user of the block to make sure that the A-compare, B-compare, and Period-compare values are in proper relation to each other.
