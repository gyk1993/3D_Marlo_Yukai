IO30X, IO31X > Driver configuration > PWM generation driver block

Dialog fields

Dialog fields

Parameter

Description

Channel vector

A vector of the PWM channels this driver block entity will access.

Channels can be specified in the range 1..8. The order of channels can be freely mixed, but the same channel shouldn't be used more than once.

Parameter group

This control selects an I/O section, the parameters of which are then shown in the dialog box below the 'Parameter group' field.

Possible groups are:

•Module setup

•Start/stop setup

•Compare count setup

•Action value setup

•Miscellaneous setup

 

 

Parameter group: Module setup

 

 

Parameter

Description

IO3XX Module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.

 

Parameter group: Start/Stop state setup

 

Parameter

Description

Initial start/stop vector

PWM generation mode values.

This vector controls whether PWM generation is in the start or stop mode. When in start mode the PWM signal is continuously generated according to the various configuration parameters. When in the stop mode the values on the A and B outputs are set to the values defined in 'Initial A/B-output stop state vector'.

The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Initial A-output start state vector

Initial values of the A outputs on entering PWM generation start mode.

 

This parameter defines the level the A output is set to when PWM generation enters start mode.

The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Initial A-output stop state vector

A vector of A-output stop state values.

The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Initial B-output start state vector

A vector of B-output start state values.

The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Initial B-output stop state vector

A vector of B-output stop state values.

The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Show start/stop input port

Enables the start/stop input port.

If this box is checked the PWM Generation block will show an input port to control the start/stop state vector at run-time (signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial start/stop vector' parameter described above.

Show A-output start state input port

If this box is checked the PWM Generation block will show an input port to control the A-output start state vector at run-time (signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial A-output start state vector' parameter described above.  

Show A-output stop state input port

If this box is checked the PWM Generation block will show an input port to control the A-output stop state vector at run-time (signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial A-output stop state vector' parameter described above.

Show B-output start state input port

If this box is checked the PWM Generation block will show an input port to control the B-output start state vector at run-time (signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial B-output start state vector' parameter described above.  

Show B-output stop state input port

If this box is checked the PWM Generation block will show an input port to control the B-output stop state vector at run-time (signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial B-output stop state vector' parameter described above.

Parameter group: Action value setup

 

Parameter

Description

Initial A-action matrix

This input port is of data type UINT8. This input port expects a matrix of size 4 x n where n must equal the number of PWM channels by the block’s 'Channel vector' parameter. Each element can be in the range from 0 to 3. The element value defines the action for each event.

The Action for the Zero Event is defined as the first element in the respective column.

The Action for the Period-Compare-Event is defined as the second element in the respective column.

The Action for the A-Up-Compare-Event is defined as the third element in the respective column.

The Action for the A-Down-Compare-Event is defined as the fourth element in the respective column.    

See the table below for descriptions of possible action values.  

Initial B-action matrix

This input port is of data type UINT8. This input port expects a matrix of size 4 x n where n must equal the number of PWM channels by the block’s 'Channel vector' parameter. Each element can be in the range from 0 to 3. The element value defines the action for each event.

The Action for the Zero Event is defined as the first element in the respective column.

The Action for the Period-Compare-Event is defined as the second element in the respective column.

The Action for the B-Up-Compare-Event is defined as the third element in the respective column.

The Action for the B-Down-Compare-Event is defined as the fourth element in the respective column.      

See the table below for descriptions of possible action values.

Show A-action input port

If this box is checked the PWM Generation block will show an input port to control the A-action input matrix at run-time (signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial A-action matrix' parameter described above.  

Show B-action input port

If this box is checked the PWM Generation block will show an input port to affect the B-action input matrix at run-time (Signal into the block). Otherwise no port for that input will be shown and the vector will be held constant during model run-time according to the 'Initial B-action matrix' parameter described above.  

 

 

Action value

Description

0

No state change (0 stays 0, 1 stays 1)

1

Clear state (0)

2

Set state (1)

3

Toggle state (0 to 1, or 1 to 0)

 

 

Parameter group: Miscellaneous setup

 

 

Parameter

Description

Trigger-output source vector

A vector of Trigger-output source values. The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

The element values must be in the range 0..3. See the table below for descriptions of possible values.

Trigger-output pulse length

A scalar value of the duration of the trigger pulse.

The value must be in the range 1..255. See the PWM code module user manual for information about the meaning of the trigger pulse and its duration.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

AB-output protection vector

A vector of AB-output protection values. The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

The element values must be in the range 0..3. See the table below for descriptions of possible values.

 

 

'Trigger-output Source' value

Condition to cause Trigger-output pulse

0

Counter value is equal to count 0

1

Counter value is equal to Period-Compare

2

Counter value is equal to A-Compare

3

Counter value is equal to B-Compare
