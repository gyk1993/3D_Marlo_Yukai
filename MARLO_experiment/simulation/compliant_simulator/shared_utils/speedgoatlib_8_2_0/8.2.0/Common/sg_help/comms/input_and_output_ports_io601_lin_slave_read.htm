Input and output ports

Input and output ports

These signals are vectors, where the vector width is set by the parameter 'Number of messages to read' (configurable through the dialog fields in this block).

 

 

Output port

Description

id

The LIN identifier received.

Range 0 to 63. Parity P0 and P1 are removed by the driver. Type uint8.

length

The data length code (DLC).

Range 0 to 8. Type uint8.

data1

Message bytes.

The LIN protocol allows messages with up to eight data bytes. Each of these outputs contains the corresponding received byte. Type uint8.

timestamp

The timestamp of when the message was received.

The resolution is 125µs.

checksum

The classic checksum of the LIN frame, based on the data only.

type

The type of the received message. Possible values:

•0: Dataframe message
•2: Error message
•3: Wakeup message

 

If 'frame type' is “Error message”,one of the following error codes is written to the 'id' output:

•0: no error
•1: bit error
•2: checksum error
•3: parity error
•4: no answer from slave
•5: Sync break error
•6: Inconsistent sync break field
•7: More data expected
•8: Timeout after start of sync break field
•9: Timeout after sync break field
•10: Timeout after sync field
•11: Slave didn’t start
•12: LIN parameter error
