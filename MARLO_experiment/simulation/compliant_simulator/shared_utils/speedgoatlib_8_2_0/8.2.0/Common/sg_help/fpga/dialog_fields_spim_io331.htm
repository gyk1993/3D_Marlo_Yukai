Dialog fields

Dialog fields

Tab: Module setup

 

 

Parameter

Description

IO3XX module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channels

Defines which SPI channels to use

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.

 

Tab: SPIM send and receive

 

 

Parameter

Description

Half period

Clock rate of the SCLK I/O line.

For example, if we define a value of [10], the serial clock line (SCLK) will be low for 10 ticks, and high for 10 ticks.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

Phase (CPHA)

The edge at which the input and output is sampled.

Options are:

•0: read on rising edge and write on rising edge

•1: write on rising edge and read on rising edge

Polarity (CPOL)

The default logic level of the SCLK I/O line when idle.

Options are:

•0: SCLK idle state is low

•1: SCLK idle state is high

Trigger duration

Time during which the CS and the SCLK are held in the trigger state.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

Post-trigger duration

Time between the end of a trigger event, and the start condition before a frame is sent.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

Start frame delay

Time between the end of a start condition, and when the frame is sent.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

Interwords delay

Idle time between words of a single frame.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

End frame delay

Time between sending the last bit and entering the stop condition state.

Units are FPGA ticks (1/33MHz ~ 30.3ns)

Bit order

Defines the bit ordering.

Options are:

•0: LSB is written first, then LSB +1, then MSB

•1: MSB is written first, then MSB-1, then LSB

 

Tab: Input and output configuration

 

 

Parameter

Description

Show RX outputs

When this box is checked, data read on the MiSo I/O line will be available in Simulink via a dedicated output port.

Enabling this checkbox introduces additional latency due to additional data transfer, therefore only check if this data is required.

Show trigger input

When this box is checked, an additional input port will be added to the send/receive driver block. This port allows you to define the duration of a trigger within a specific frame.
