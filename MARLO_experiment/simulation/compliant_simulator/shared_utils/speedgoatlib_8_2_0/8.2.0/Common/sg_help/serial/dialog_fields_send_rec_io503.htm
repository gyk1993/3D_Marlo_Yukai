Dialog fields

Parameter

Description

Parameter group

This control selects an I/O section, the parameters of which are then shown in the dialog box below the 'Parameter group' field.

 

Possible groups are:

•Board Setup

•Basic Setup

•FIFO Setup

 

 

Parameter group: Board Setup

 

 

Parameter

Description

IRQ number

Serial communication is managed as an asynchronous event. The driver block therefore requires information about the IRQ source for the related I/O module. IRQ information can be found in the technical reference document provided together with your Speedgoat real-time target machine. Information is also available by typing the following at the MATLAB command prompt: getxpcpci.

PCI Slot (-1:autosearch)

If only one module of this type is present in the target machine, leave as default ( '-1') to automatically locate the board. If more than one module of this type is present, enter the bus number and the PCI slot number of the board associated with this driver block using the format '[BusNumber, SlotNumber]'. To determine the bus number and the PCI slot number, type 'getxpcpci' in the MATLAB command window

 

 

Parameter group: Basic Setup

 

Parameter

Description

Port to modify

The serial port you to configure using the parameters below.

Transceiver setup

Configuration of the transceiver. It represents the physical layer of the serial protocol. Options are:

•RS232 (max 921.6 kb/s)

•RS422

•RS422 multidrop

•RS485 full-duplex

•RS485 half-duplex

Use standard baud rate (16x, no prescaling)

Check this box if you want to use a standard baud rate
(listed in the 'Standard baud rate' parameter below)

Standard baud rate

A list of standard baud rate (only used if the 'Use standard baud rate' checkbox is ticked)

Baud rate divisor (non-standard baud rates)

Non-standard baud rate (only used if the 'Use standard baud rate' checkbox is not ticked)

The non-standard baud rate is determined by the following formula:

Baud rate = 44.2368MHz / (Sampling rate * Divisor * Prescale)

 

Sampling rate is set in the dialog box and is either 16 (normal operation) or 8.

Prescale = 1 if “Prescale by 4” option in the dialog box is not checked

Prescale = 4 if “Prescale by 4” is checked

Sampling rate (non-standard baud rates)

For non-standard baud rates, the sampling rate can be set to 8x or 16x (normal operation).

Transmit and receive data rates will double by selecting 8x sample rate.

Prescale by 4 (non-standard baud rates)

Sets the Prescale value used to calculate a non-standard baud rate.

See the 'Baud rate divisor' parameter above for information on how the value is used.

Parity

Parity is a method for detecting errors during transmission. Options are:

•None

•Even

•Odd

•Mark

•Space

Data bits

The number of data bits for each character. Options are: 5, 6, 7 or 8.

Stop bits

The number of stop bits sent at the end of every character. Options are: 1 or 2.

Receive FIFO interrupt level

This parameter specifies the number of characters in the receive FIFO before an interrupt occurs. Receive interrupts occur at least as often as this parameter specifies.

If a gap of at least 4 character periods occurs in a data stream, the UART requests an interrupt for the receiver. The UART requests an interrupt regardless of the value of the 'Receive FIFO interrupt level'. If the hardware FIFO contains at least one character, an interrupt is signaled.

Options are:

•1

•Quarter full (16)

•Half full (32)

•Almost full (56)

Hardware handshake

Automatic hardware RTS/CTS flow control is used to prevent data overrun to the local receiver FIFO and remote receiver FIFO.

Options are:

•RTS/CTS

•None

Software handshake

Software flow control. When enabled, the XOFF character is transmitted approximately two character time periods after the RX FIFO level has reached the RX trigger level. The XON character is transmitted when the RX FIFO level falls below the RX trigger level.

Options are:

•XON/XOFF

•None

XON character

If 'Software handshake' is enabled, this parameter defines the XON character.

XOFF character

If 'Software handshake' is enabled, this parameter defines the XOFF character.

RS485 turnaround

This check box enables RS-485 automatic turnaround.

In RS-485 mode, where the transmitters and receivers use the same differential pair of wires, you can only enable one transmitter at a time. Select this check box to turn on the automatic transmitter control in the UART. The RTS output is routed to the enable input of the RS-485 transmitter.

RS485 turnaround delay [0,15]

The desired RS-485 turnaround delay (if 'RS485 turnaround' is enabled). The UART automatically asserts RTS when there are characters in the transmit FIFO. It de-asserts RTS when the transmit FIFO empties and the RS-485 turnaround default delay of 0 to 15 bit time periods elapses. The transmitter output goes tristate (high impedance) when disabled, which allows another device to transmit.

 

 

Parameter group: FIFO Setup

 

 

Parameter

Description

Port to modify

The serial port to be configured by the parameters below.

Transmit software FIFO size

The size of the software FIFO used to buffer transmitted characters.

Transmit FIFO data type

The data type of the transmitter.

Options are:

•Count+32 bit int

•Count+32 bit uint

•Count+16 bit int

•Count+16 bit uint

•8 bit int null terminated

•8-bit uint null terminated

8-bit data types require a NULL-terminated string in the input vector.

16- and 32-bit data types reserve the first full element for holding the number of elements to expect in the rest of the input vector. Only the low-order byte of each data element is sent. Setting this data type allows a wider data type to hold the bytes. If the data stream needs to include a NULL byte, you must select one of the16- or 32-bit data types.

Receive software FIFO size

The size of the software FIFO to buffer characters between interrupt service and periodic execution.
