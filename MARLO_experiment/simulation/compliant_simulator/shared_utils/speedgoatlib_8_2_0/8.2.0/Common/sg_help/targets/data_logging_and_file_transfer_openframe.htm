Data logging and file transfer

Simulink Real-Time provides functionality to log data during real-time execution on the main drive of your target machine. Further information on how to configure and use the Simulink Real-Time software for data logging is provided in the Simulink Real-Time User’s Guide from The MathWorks.

 

To access and transfer logged data from the host to the target and vice versa you can use:

 

•Simulink Real-Time File System, using MATLAB commands
•A USB drive and DOS commands on your target machine

 

Important Note

 

Do not change or delete DOS files saved on the main drive of your target machine. If you accidentally remove or modify such files contact Speedgoat for recovery instructions.

 

MATLAB commands

 

To transfer a file from the target to the host:

 

SimulinkRealTime.copyFileToHost('filename.xyz') 

 

The file is stored in the current MATLAB workspace.

 

Additional File System commands are described in the Simulink Real-Time documentation.

 

USB drive and DOS commands

 

If your target machine has a PS/2 keyboard port, you can use DOS commands to transfer files between the provided USB drive and the target main drive.

 

For this method you need a monitor and a PS/2 keyboard connected to the target machine.

 

The USB drive must contains the "dos.sg" file. If the file doesn't exist, create a new empty text file and rename it to "dos.sg".

 

Insert the USB drive into a USB port of your target machine which supports kernel transfer (refer to the technical specification of your system). Once you power up or reboot the target machine, the boot process stops at the DOS command prompt, where you can use standard DOS commands.

 

To copy a file from the target machine main drive C to the USB drive (usually drive D):

 

copy C:\filename.xyz D:\

 

Important note

 

Not all USB drives are supported by the provided DOS USB driver. Speedgoat therefore strongly recommends to use the USB drive provided to you as part of the target machine delivery.

 

Log data in an external USB drive

 

Your target machine includes a fixed HDD or SSD, depending of ordering options. However, you can store data during model execution in an external USB memory drive, you can connect it to a free USB port of your target machine, before powering up the system.

 

If you want to use the provided USB drive, please remove the "dos.sg" file from it.

 

Usually the external USB drive will take drive letter D. You can check that in Simulink Real-Time Explorer. To store data in an external device, please refer to the Simulink Real-Time documentation.
