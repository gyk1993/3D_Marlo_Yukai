IO750 > Driver configuration > Setup driver block

Dialog fields

Dialog fields

Tab: Configuration

 

 

Parameter

Description

IO75X module identifier

As it's possible to have more than one IO75X module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one IO75X module then leave the default value (1).

Target machine drive letter

The drive letter of your target machine (default “C:”)

Configuration mode

Select “Configuration file” to use configuration files generated with SYCON.net.

Use “Warmstart” to configure the EtherCAT slave with the parameters defined in the 'Warmstart parameters' tab (see below).

Sample time [s]

Defines how often this block is updated.

PCI slot (-1: autosearch)

If only one module of this type is present in the target machine, leave as default ( '-1') to automatically locate the board. If more than one module of this type is present, enter the bus number and the PCI slot number of the board associated with this driver block using the format '[BusNumber, SlotNumber]'. To determine the bus number and the PCI slot number, type 'getxpcpci' in the MATLAB command window

 

Tab: Warmstart parameters

 

Parameter

Description

Input data bytes (Output for the Master, TxPdo)

The number of bytes to receive from the Master. In SYCON.net this parameter is named “Output Data Bytes” because it refers to the Master side.

Output data bytes (Input for the Master, RxPdo)

Insert the number of bytes to send to the Master. In SYCON.net this parameter is named “Input Data Bytes” because it refers to the Master side.
