Embedded option

You can use Speedgoat real-time target machines with the xPC Target Embedded option. This requires an xPC Target Embedded license.

 

The embedded option allows the integration of your application directly in the kernel image, for deployment. As soon as the kernel is loaded, it will automatically start the application.

 

To configure, generate and load a embedded kernel image on a Speedgoat target machine, proceed as follows:

 

1.Open xPC Target Explorer, by typing xpcexplr.

 

2.In the Boot configuration section, select 'Stand Alone' as boot mode.

 

3.Save the configuration and exit xPC Target Explorer.

 

4.Build your Simulink model. The build process generates the stand-alone target application into subdirectory "modelname_xpc_emb".

 

5.In the subdirectory "modelname_xpc_emb" locate the "modelname.rtb" file. This is the embedded kernel image.

 

6.Insert the provided USB memory device and copy the "modelname.rtb" file on to it.

 

7.Rename the "modelname.rtb" file to "xpckrnl.rtb".

 

8.Make sure that the USB memory device contains the "dos.sg" file. If the file doesn't exist, create a new empty text file document and rename it "dos.sg".

 

9.Insert the USB memory device in a USB port of your target machine which allows kernel transfer (refer to the technical specification of your system).

 

10.Power-up or reboot your target machine. The Speedgoat kernel transfer tool will detect and copy the embedded kernel image to the main drive.

 

11.Follow the instructions on the target screen, remove the USB memory device and reboot the system.

 

The kernel image will load and start automatically on the target machine. You can control the application from MATLAB and xPC Target Explorer.
