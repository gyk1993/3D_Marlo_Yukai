This control selects an I/O section, the parameters of which are then shown in the dialog box below the 'Parameter group' field. Possible groups are:

 

•Module setup

Parameter group: Module setup

 

 

Parameter

Description

IO3XX Module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

FPGA bitstream

The FPGA bitstream implementing the defined set of FPGA Code Modules to download to the FPGA chip of your IO3XX I/O module.

The name of the default FPGA bitstream provided with the IO3XX is speedgoat_IO3XX_default (e.g. for the IO302 module use filename speedgoat_IO302_default)

The FPGA bitstream will be downloaded onto the FPGA chip of the IO3XX I/O module during the build process (C code generation of Simulink model using Simulink Coder and download on target machine).

PCI Slot (-1:autosearch)

If only one module of this type is present in the target machine, leave as default ( '-1') to automatically locate the board. If more than one module of this type is present, enter the bus number and the PCI slot number of the board associated with this driver block using the format '[BusNumber, SlotNumber]'. To determine the bus number and the PCI slot number, type 'getxpcpci' in the MATLAB command window

 

 

Parameter group: Lower channel group setup

 

 

Parameter

Description

Direction vector

The direction vector allows a channel pair (two lines) to be defined as input or output.

 

The first scalar defines the direction for I/O lines 1 and 2, the second for I/O lines 3-4, and so forth (see also I/O pin mapping). A value of 0 defines the channel pair as an input, and a value of 1 defines the channel pair as an output.

 

If a scalar value is provided then scalar-expansion applies. So a scalar value of [0] defines all channel pairs as inputs, and a scalar value of [1] defines all channel pairs as outputs.

 

The configuration of this parameter depends on the module and the bitstream used. See the section 'Lower channel direction vector' for example configurations.

Reset vector

This field specifies the behaviour of the channels at model termination.

 

Enter a scalar or a vector of the same length as the channel vector. If a scalar value is specified, its value will be used for all channels. A value of 1 causes the corresponding channel to be reset to the value specified in the 'Initial value vector' field. A value of 0 causes the channel to remain at the last value used while the model was running.

 

If a vector is provided then its width must be 16 (8 for IO303 and IO313). The elements of the direction vector which are set to 0 (inputs) won’t be affected by the value defined in here.

 

Alternatively the reset behavior of each channel defined as output can be set individually.

Description

The vector width must be 16 (24 for IO303 and IO313).

 

If a scalar value is provided then scalar-expansion applies. So a scalar value of [0] defines all channel pairs as inputs, and a scalar value of [1] defines all channel pairs as outputs.

 

The configuration of this parameter depends on the module and the bitstream used. See the section 'Upper channel direction vector' for example configurations.
