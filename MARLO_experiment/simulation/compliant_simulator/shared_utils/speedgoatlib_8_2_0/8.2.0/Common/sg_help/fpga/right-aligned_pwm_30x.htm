Right-aligned

Here is a timing diagram for a right-aligned (asymmetrical) signal:

 

 

The counter increments until the half-period (C) and then decrements to zero again.

The line A signal goes high on the A-up event and low on the zero event.

The line B signal goes high on the B-up event and low on the zero event.

The signals are therefore left aligned.

 

In order to define at which points on the counting cycle the polarity of the signal changes the four 'A-action' elements are configured as in the following table.

'A-action' is either set by the 'Initial A-action matrix' parameter, or by the 'A-action' input port.

 

Event (see Events sub-section)

Configuration value

Zero

1 - Clear state

Period-Compare

0 - No state change

A-Up-compare

2 - Set state

A-down-compare

0 - No state change

 

So for this example the A-action matrix would be [1,0,2,0]'.

 

The timing of the A-Up and A-Down events is defined by the 'A-compare count' input port (initialized by the 'Initial A-compare start state vector' configuration parameter).

 

The B signal is similarly configured.

 

The timing of the C event (the point when the counter begins counting down to zero, half of the period T), is defined by the 'Period-compare Count' and initialized by the 'Initial period-compare start state vector' configuration parameter. See section 'Setting compare values' for information on calculating the PWM output frequency.
