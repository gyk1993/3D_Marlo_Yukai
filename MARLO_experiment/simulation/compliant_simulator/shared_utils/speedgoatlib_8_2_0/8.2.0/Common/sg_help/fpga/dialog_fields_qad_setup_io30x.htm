Dialog fields

 

Parameter

Description

IO3XX module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

A vector of QAD channels this driver block entity will access. You can specify channels in the range 1..N. The order of channels can be freely mixed, but the same channel shouldn't be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Please observe that all the channels will have the same configuration. In the case you need to have a different configuration for others channels, you have to use an additional QAD setup block in your model.

Quadrature mode

This parameter changes the counting mode. Possible values are:

•1x: counter increments or decrements at each rising edge event on line A

•2x: counter increments or decrements at each rising and falling event on line A

•4x: counter increments or decrements at each rising and falling event on line A and B

This parameters affects the minimal and maximum range per revolution. For example if you select “1024” as steps per revolution and “4x” as quadrature mode then the revolution range will be: [-4096, 0, 4095].

Index control mode

Defines the action for each C/Index event detection. Possible values are:

•Ignore index input: no action

•Load on index: Counter value is set to the init val signal

•Latch on index: Current counter value is stored into the position register, and the position register will only be updated at the next C/Index event.

•Reset on index: Counter resets to zero

•Index reference mode: Counter is reset to zero at the first C/Index event detection and Index control mode then changes to “Ignore index input”

Special count mode

Defines the minimum and maximum counting value. Possible values:

•Disabled (Cycling counter): The counter runs from -2^31 to 2^31 – 1. Its current value divided by the product of 'Steps per revolution' and 'quadrature mode' gives the number of revolutions done.

•Divide-by-N: In this count mode, the counter is modulo the product 'Steps per revolution' and 'quadrature mode'.

•Single cycle: counter is disabled when the absolute limit is reached. To re-enable the counter the position register(s) need to be reset with the counter command signal.

Steps per revolution

The resolution of your physical encoder. Typical values are powers of two such as: 1024, 2048 or 4096.

Sample time

The base sample time or a multiple of the base sample time at which the instance of the Write driver block will execute. A sample time of -1 defines inherited sample time deduced from the driving block.
