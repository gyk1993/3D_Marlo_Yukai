IO331 > Driver configuration > EnDat

Dialog fields

Dialog fields

Tab: Module setup

 

 

Parameter

Description

FPGA module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

A vector of EnDat channels this driver block entity will access. Channels can be specified in the range 1..N. The order of channels can be freely mixed, but the same channel shouldn't be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.

 

 

Tab: EnDat configuration

 

 

 

 

Parameter

Description

Automatic detection of encoder data width

If checked, the EnDat driver block will automatically determine the data width configuration of the encoder by reading out its internal registers.
If unchecked, the data width configuration can be configured manually.

position data width vector

Configures the number of data bits for the position reading of the EnDat encoder.
Valid range 1 to 32.

distinguishable revolutions data width vector (multiturn only)

Configures the number of bits for the distinguishable revolutions reading of the EnDat encoder.
For single-turn encoder a value of 0 needs to be configured.
Valid range0 to 32.

transmission frequency factor vector

Configures the EnDat transmission frequency.
The resulting transmission frequency can be calculated as follows:
f_trans = f_fpga / (2 * transmission frequency factor)

recovery time vector

Configures the recovery time (tst) for start bit recognition. With low transmission rates (EnDat transmission frequency f_trans < 400kHz) the same value as "transmission frequency factor" can be set. With high transmission rates, the recovery time need to be set much higher than the frequency factor.

Unit is in ticks*.

Show ErrorBit port

If checked, the error bitfield port is enabled.

The error bitfield port holds information of several errors decoded as follows:
- Bit 0: F-Typ III Error 1F
- Bit 1: F-Typ III Error 0F
- Bit 2: F-Typ II  Error
- Bit 3: F-Typ I   Error
- Bit 4: CRC Error
- Bit 5: Error bit 2 of EnDat protocol (Alarm2)
- Bit 6: Error bit 1 of EnDat protocol (Alarm1)
- Bit 7: FPGA configuration error

 

*        For IO30X and IO31X: 1 tick = 1/33Mhz (FPGA frequency of 33MHz)

For IO331: 1 tick = 1/75MHz (FPGA frequency of 75MHz)
