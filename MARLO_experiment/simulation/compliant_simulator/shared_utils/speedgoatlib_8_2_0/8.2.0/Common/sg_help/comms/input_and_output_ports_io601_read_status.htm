IO601 > Driver configuration > CAN > Read status CAN port driver block

Input and output ports

Input and output ports

Input port

Description

Status CAN Port

This is used to read the 'CAN Port status' output from the Status block

This signal is a scalar of type uint16.

 

 

Output port

Description

Error/Warning

A value of 1 signifies that the CAN port reached an Error/Warning level.

This signal is a boolean.

Bus Off

A value of 1 signifies that the CAN port reached the Bus Off level.

This signal is a boolean.

Data Overrun

A value of 1 signifies that the CAN port‘s transmit queue has overrun.

This signal is a boolean.

Transmit

A value of 1 signifies that the firmware is currently transmitting a CAN message.

This signal is a boolean.

Transceiver Error

A value of 1 signifies that the firmware detected a transceiver error. This is only checked for Lowspeed transceiver mode.

This signal is a boolean.

Receive Queue Overrun

A value of 1 signifies that the CAN port‘s receive queue has overrun.

This signal is a boolean.

Receive Queue Empty

A value of 1 signifies that the CAN port‘s receive queue is currently empty.

This signal is a boolean.

Receive

A value of 1 signifies that the firmware is currently receiving a CAN message.

This signal is a boolean.
