Parameter

Description

Parameter group

This control selects an I/O section, the parameters of which are then shown in the dialog box below the 'Parameter group' field. Possible groups are:

•Module setup

•Digital input setup

•Digital output setup

 

 

Parameter group: Module setup

 

 

Parameter

Description

IO204 module identifier

As it's possible to have more than one IO204 module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one IO204 module then leave the default value (1).

PCI Slot (-1:autosearch)

If only one module of this type is present in the target machine, leave as default ( '-1') to automatically locate the board. If more than one module of this type is present, enter the bus number and the PCI slot number of the board associated with this driver block using the format '[BusNumber, SlotNumber]'. To determine the bus number and the PCI slot number, type 'getxpcpci' in the MATLAB command window

 

 

Parameter group: Digital input setup

 

 

Parameter

Description

Enable debounce

This setting applies to all 32 input channels. If unchecked inputs are not debounced. If checked the inputs are debounced based on the 'Debounce time' setting below.

Debounce time [s]

This setting applies to all 32 input channels. If the 'Enable debounce' dialog field (see above) is checked then input changes faster than the time specified here will be ignored. The debounce time is the same for all input channels and can be specified in seconds in the range of 4e-6s (4us) and 0.440s (440ms) in steps of approximately 7us.

 

 

Parameter group: Digital Output setup

 

 

 

Parameter

Description

Reset vector

This field specifies the behaviour of the channels at model termination.

 

Enter a scalar or a vector of the same length as the channel vector. If a scalar value is specified, its value will be used for all channels. A value of 1 causes the corresponding channel to be reset to the value specified in the 'Initial value vector' field. A value of 0 causes the channel to remain at the last value used while the model was running.

Initial value vector

This field specifies the behavior of the channel at model termination. Enter a scalar or a vector of the same length as the channel vector. If a scalar value is specified, its value will be used for all channels. A value of 1 causes the corresponding channel to be reset to the value specified in the 'Initial value vector' field. A value of 0 causes the channel to remain at the last value used while the model was running.
