Basic model

Create a new Simulink model in your workspace and name it "rtUDP_1.slx". Configure the model for Simulink Real-Time:

 

1.Simulation > Model Configuration Parameters (or ctrl-E)
2.Select Solver, set the Stop time to 'inf'
3.In the Solver options panel, set Type to 'Fixed-step' and Solver to 'discrete (no continuous states)'
4.Select Code Generation, set System target file to 'slrt.tlc' (use browse)
5.Click OK

 

From the xpcrtudplib library, drag and drop the "Network Configuration" block into the model.

 

Note: You may also use the Speedgoat libraries (e.g. Network Configuration LAN1). The blocks are interchangeable.

 

 

Network configuration block parameters

Select a valid IP and subnet mask for your network. In this example, the following setup has been used:

 

TARGET        IP address:        192.168.7.1

HOST                IP address:        192.168.7.2

         Subnet mask:        255.255.255.0

 

The next available IP 192.168.7.3 is used for the IO710/IO713 I/O module. Insert the PCI bus and slot information according to the setup of your target machine. Refer to the section on 'PCI bus and slot numbers' (in the 'Setting up' section) for help on getting this information. We use 1 second as the sample time in this model.

 

Network Configuration block parameters

 

 

Save and build the model. As soon as you start the model on the target machine, the IP address is being assigned to the selected IO710/IO713 I/O module.

 

You may connect the Ethernet cable used for Host/Target communication to the IO710/IO713 Ethernet port. Pinging the target from the host (using a DOS Prompt) leads to time outs:

 

 

However, by issuing the command “arp –a”, the correct mapping is displayed at the DOS shell

 

 

This is a simple way to determine the MAC address of your IO710/IO713 Ethernet port.
