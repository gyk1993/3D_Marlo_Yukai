About the RS422 protocol

RS422 is a serial communication protocol where data is sent one bit at a time, sequentially, over a pair of wires. RS422 provides a mechanism by which serial data can be transmitted over long distances (up to 1500m) and fairly high rates. This is accomplished by splitting each signal across the two different wires in opposite states.

The difference in voltage between the two wires is compared by a receiver to determine the logic state of the signal. In this way if there is a common noise signal on both wires it will be ignored. Errors can however be introduced by noise on a single wire. To help reduce the chance of this, each receiver is grounded:

 

A single driver can drive a transmission line with up to 10 receivers. The common-mode voltage (VCM) is defined as the mean voltage of the A and B terminals with respect to signal ground. RS422 receivers can withstand a common-mode voltage of +/-7V.

Since one master can communicate with up to 10 devices, a mechanism for controlling data exchange must be implemented. For this purpose two mechanisms are available:

•Half-Duplex: communication can only take place in one direction at a time.

•Full-Duplex: transmission and reception can occur at the same time.

 

The message frame format can be built in different ways, but typically is configured as follows:

•Header: code consisting of one byte is for frame synchronisation

•ID: equipment ID number.

•Length: sum of bytes from header to the end.

•Data: data sent or received

•Checksum: byte assuring data integrity

•End: byte defining the end of a frame

All blocks are sequences of 8 bits (byte), and the least significant bit (LSB) is sent first.

A start bit indicates the start of a bit sequence and is detected when a transition occurs from high to low. Eight data bits follow the start bit, followed by a stop bit which is one bit long. Another start bit starts the next bit sequence. When the last character is sent the line should stay high until the next start bit.
