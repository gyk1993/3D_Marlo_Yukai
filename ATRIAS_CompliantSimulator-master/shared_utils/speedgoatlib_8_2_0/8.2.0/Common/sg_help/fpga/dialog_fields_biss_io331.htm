Dialog fields

Dialog fields

Tab: Module setup

 

 

Parameter

Description

FPGA module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

A vector of BiSS channels this driver block entity will access. Channels can be specified in the range 1..N. The order of channels can be freely mixed, but the same channel shouldn't be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.

 

Tab: BiSS configuration

 

 

 

 

Parameter

Description

Clock divider vector

Configures the BiSS transmission clock rate.

E.g. if we define a value of [10], the clock line will be low for 10 ticks, and high for 10 ticks.
Units are in ticks*.

Valid range 6 to 468, which corresponds to a frequency range of approx. 6.25Mhz to 80kHz.

Time out time vector

Configures the BiSS driver block time out time.
This is mainly used to reset the BiSS slaves (common times are specified to 40us, which corresponds to 3000 ticks*).

Data bits vector

The number of BiSS data bits.

Additional transmission bits like Warning, Error and CRC are configured separately.

Valid range 1 to maximum 63 (depends on the additional flag settings).

Error Flag

Configures a Error flag port.

Warning Flag

Configures a Warning flag port.

CRC Bits

Configures the CRC bits port.

Number of CRC Bits

Configures the number of CRC Bits.
Valid range 0 to 8.

Show transmission error

If checked, enables the transmission error port. This port will indicate (value == 1) if something went wrong during the BiSS data transmission.

 

 

*                For IO30X and IO31X: 1 tick = 1/33Mhz

For IO331: 1 tick = 1/75Mhz
