Dialog fields

 

Parameter

Description

IO3XX module identifier

As it's possible to have more than one reconfigurable FPGA module in a target machine, this parameter is used by the other driver blocks to identify a specific module. Select a unique number (1..10) for each of your modules. If your target machine only contains one FPGA module then leave the default value (1).

Channel vector

A vector of CAP channels this driver block entity will access. Channels can be specified in the range 1..8. The order of channels can be freely mixed, but the same channel shouldn't be used more than once. The width of this vector also defines the resulting size of some of the following parameters if scalar expansion applies.

Start event vector

A vector of Start events (events that trigger the starting of a measurement).

Use a value of 0 for a high-to-low transition and value 1 for a low-to-high transition. The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Stop event vector

A vector of Stop events (events that trigger the ending of a measurement).

Use a value of 0 for a high-to-low transition and value 1 for a low-to-high transition. The width of this vector must be equal to the width of the 'Channel vector' parameter.

Alternatively, if a scalar value is specified, its value will be used for all channels.

Period vector

An optional upper limit on the measurement value.

 

After this time if the Stop event has not triggered, the measurement will automatically finish and a new one will begin immediately.

 

Set to 0 to disable (default).

 

See the Usage Notes for this module for more information on the use of this parameter.

Sample time

Defines the base sample time at which this driver block gets its sample hit. It can also be set to -1 for inherited sample time.

 

Units are seconds.
