Input and output ports

These signals are vectors, where the vector width is set by the parameter “Number of messages to read” (configurable through the dialog fields).

 

 

Output port

Description

id

The LIN identifier received.

Range 0 to 63. Parity P0 and P1 are removed by the driver. Type uint8.

length

The data length code (DLC).

Range 0 to 8. Type uint8.

data1

Message data.

The LIN protocol allows messages with up to eight data bytes. Each of these outputs contains the corresponding received byte. Type uint8.

timestamp

The timestamp of when the message was received.

Type uint8. The resolution is 125µs.

checksum

The classic check sum of the LIN frame, based on the data only.

Type uint8.

type

This port can take three possible values:

•0: Dataframe message
•2: Error message
•3: Wakeup message

 

In case, the frame type is “Error message”, the error code is written to the 'id' output. The error codes are:

•0: no error
•1: bit error
•2: checksum error
•3: parity error
•4: no answer from slave
•5: Sync break error
•6: Inconsistent sync break field
•7: More data expected
•8: Timeout after start of sync break field
•9: Timeout after sync break field
•10: Timeout after sync field
•11: Slave didn’t start
•12: LIN parameter error

 

Type uint8.

info

This field contains 3 bits of message information.

•Bit 0: 0 signifies no overrun, 1 signifies overrun
•Bit 1 : 0 signifies message received, 1 signifies message sent (self-reception)
•Bit 2: 0 signifies identifier and data, 1 signifies only identifier

In case of a self-reception, this output will be equal to 2 (bit 1 set).

Type uint8.

 

Input port

Description

id

The LIN identifier to read.

Range 0 to 63. The parity is computed by the driver. Type uint8.

length

The data frame length code. It specifies how many bytes the master expects to receive.

When the id and the length input are set to zero (0), the S-Function interprets this as “don’t send the header”.

In order to configure a loop back on a single board:

•A LIN master write block must be deployed in the model
•The LIN master read block identifier and length must be tied to 0.
•The LIN master read block will retrieve the data sent by the master (self-reception).

 

Type uint8.
