1 Free of charge vs. Professional

Free of charge compilers are perfectly adequate for simple to moderately complex models. However for large and complex models our tests have shown that the additional optimizations of the 2010 and 2012 edition professional compilers can reduce task execution time by up to 40-50%. The performance gains for the 2008 edition have less impact.

For previous releases please visit: www.mathworks.com/support/compilers/previous_releases.html.

 

Set the compiler

 

From R2013b, the mex -setup command sets the default compiler for xPC Targets builds, provided that the MEX compiler is supported by MathWorks. Use (xpcsetCC -setup for pre R2014a) only if you have to specify different compilers for MEX and xPC Target.

 

For previous MATLAB releases, MEX and xPC Target compilers must be specified.

 

Use mex -setup to select the MEX compiler.

 

Use xpcsetCC -setup to set the xPC Target C-compiler.

 

This command is available from R2011a. For R2010b or previous release, use the CompilerPath property, configurable in xPC Target Explorer. Type slrtexplr (xpcexplr for pre R2014a) at the to open the xPC Target Explorer, and navigate to Compiler(s) Configuration. At the 'Select C Compiler' drop-down list select the compiler you have installed e.g. VisualC. Then navigate to 'Compiler Path' and enter the path to the compiler executable file (.exe). Click Apply to apply settings.  

If the compiler doesn't appear on the list, please verify that the compiler is correctly installed.

 

For older MATLAB releases, you can use the command setxpcenv('CompilerPath','C:\Program Files (x86)\Microsoft Visual Studio 10.0') to manually select the xPC Target compiler path.

To verify that the xPC Target compilers path is configured correctly, use the command getxpcenv(‘CompilerPath’).

 

 

Installing Microsoft Windows SDK 7.1 in Windows 8

 

If you already have a version of the Visual C++ 2010 Redistributable installed on your computer, you may need to uninstall it first before installing the SDK.

For more information see: http://support.microsoft.com/kb/2717426

 

 

Upgrading from Microsoft Visual C++ 2010 Service Pack 1 to Microsoft Windows SDK 7.1

 

New users of the Microsoft Windows SDK 7.1 upgrading from the Microsoft Visual C++ 2010 Service Pack 1 might experience error messages during installation. To avoid please upgrade using the below described process.

 

1.If you have already installed the Windows SDK, please uninstall it.
2.Install the Windows SDK without Visual C++ Compilers.
3.Download and install the “Microsoft Visual C++ 2010 Service Pack 1 Compiler Update for the Windows SDK 7.1” patch available at http://www.microsoft.com/en-us/download/details.aspx?displaylang=en&id=4422.
