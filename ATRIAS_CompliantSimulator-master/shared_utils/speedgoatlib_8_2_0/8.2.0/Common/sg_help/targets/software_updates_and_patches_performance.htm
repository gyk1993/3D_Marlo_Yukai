Software updates and patches

Software updates and patches

MathWorks provides various patches and bug fixes. Usually all improvements are included in the next MATLAB release. Please take a look at the MathWorks support page: http://www.mathworks.com/support/bugreports/.

 

Speedgoat usually provide you with all patches which are related to your target machine or installed I/O modules, however here you can find references for common problems.

 

Intel i3/i5/i7 CPU support

 

Kernel patch for third generation Intel i3/i5/i7 CPUs (for R2012b – R2013b, fixed in R2014a).

 

Please contact us to get the patch.

 

http://www.mathworks.com/support/bugreports/970719

 

EtherCAT Master

 

EtherCAT in xPC Target does not work correctly with the newest TwinCAT or ET9000 configuration tools (for R2011b - R2012b, fixed in R2013a).

 

http://www.mathworks.com/support/bugreports/834038

 

Nonstandard EtherCAT data types are not handled correctly (for R2011b - R2013a, fixed in R2013b).

 

http://www.mathworks.com/support/bugreports/932059

 

File scopes

 

xPC kernel crashes while executing a model with file scopes, dynamic file mode enabled, and autorestart disabled (for R2010b - R2012a, fixed in R2012b).

 

http://www.mathworks.com/support/bugreports/805812

 

Real-Time UDP

 

Real-Time UDP Send and Receive blocks do not honor the "Use following local IP port" and "IP Address to receive from" settings, respectively (for R2011a - R2012a, fixed in R2012b).

 

http://www.mathworks.com/support/bugreports/802074

 

Serial ports and serial I/O modules

 

Incorrect Code Generation: RS-232 blocks fail with transmit FIFO overflow message (for R2012a, fixed in R2012b).

 

http://www.mathworks.com/support/bugreports/826451

 

xPC Target Explorer

 

When you launch xPC Target Explorer, it fails to open and produces the error message: "Object reference not set to an instance of an object" (for R2012b, fixed in R2013a).

 

http://www.mathworks.com/support/bugreports/874378
